A SaaS Application to record timesheets, time off, and expenses

The Idea

We were approached by a client who were unhappy with their current system provider. They wanted us to create a new Timesheet and Time Off system for them. Their current system did not allow for them to track expenses. This was a module that our company felt was a natural fit to add. In addition, we wanted to make the system so it could be used by other companies should the oppurtunity arise.

Being the only Web Developer at the firm, I was in charge of the project. I hand coded the application using HTML, CSS, JS, PHP, and MySQL. A lot of the technology and techniques were was new to me, but I learned a lot along the way, and delivered a product that the customer was overjoyed with.

Challenges

Server Setup

The first challenge came with the server setup. Unlike most websites I had worked on in the past, Synkros was not suited for shared hosting. I need to have full control over the server so that I could easily configure it to fit my needs. The biggest need for this was to configure the virtual host file. The idea was that since this was intended to be a SaaS application, each client would have their own subdomain. To accomplish this, I needed to configure Wildcard hosts.


Structure and Administration

After a lot of research, I found the best way to develop the application was to have one codebase, and a database for each client. This would allow for easier updates to the code, while also keeping the data seperate for each client.

To create new clients, I developed an admin portal. The admin portal allowed me to easily create a new client by filling out a form. Once the form was filled out, a database and database user were created for the client. By creating a database user for the clients with only the necessary privileges, the system was more secure.


Database Design and MySQL

Before this project, I only had a little experience with MySQL. To make Synkros as dynamic as possible, the code relied heavily on the database. The database was designed to be flexible, to allow Synkros to be as dynamic as possible. By allowing different settings to be stored in the database, it would allow different clients to use the system differently.

At the start of the project, MySQL was very new to me. I learned a lot of SQL along the way. The biggest challenge was allowing MySQL events to be set and altered on a per client basis. It was a great learning experience and really strengthened my knowledge of MySQL.

Development

The development process included both front-end and back-end work. The UI was created with HTML, CSS, JavaScript, and jQuery. To save time on development, we purchased a template, which I then altered to be more altered to use PHP. Cross browser compatability was something I needed to stay aware of as well. I fixed multiple issues where Synkros didn't work properly across browsers.

For version control, I utilized GitHub. Synkros went through extensize testing before deploying to clients. I made sure that any possible hole in the program was fixed before deploying. However, if I client ran into an issue, version control easily allowed us to rollback to a stable build

Conclusion

Synkros was a huge learning process for me. It allowed me to develop my PHP and MySQL knowledge, as well as get a further understanding of the front-end aspect. Creating a client facing application gave me an appreciation of how much work goes into an application. It showed me the importance of creating an intuitive application and how important extensive QA is. Creating a client facing application from start to finish is something I am extremely proud of. I learned a lot during the development process while meeting deadlinesThis project really pushed me to become a better developer. This experience will allow me to create even better applications in the future.
